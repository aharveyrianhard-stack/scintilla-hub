2 · Methodology — the model math

CoE (CAPM) = rᶠ + ( β × MRP )
WACC = ( wₑ × CoE ) + ( w_d × k_d × ( 1 − t ) )  // NVDA wₑ=99.8%, w_d=0.2%
UFCFₜ = EBITₜ × (1 − t) + D&Aₜ − capexₜ − ΔNWCₜ
Terminal value = ( FCF₅ × ( 1 + g ) ) ÷ ( WACC − g )  // g = perpetuity growth, distinct from rᶠ, must be < WACC
EV = Σ PV(UFCF₁…₅) + PV(terminal)  ·  Fair value = ( EV − net debt ) ÷ diluted shares

Base facts (NVDA): FY2026 revenue $215.9B · 24.51B diluted shares · net debt $0.8B · fixed-from-history: D&A 3.06%, capex 3.49%, NWC 22.7% of revenue.

Reverse DCF

Lock every input at the current selection except one; bisection-solve that one so fair value = market price ($202.81). The result is the market's implied assumption. The 2-var grid does this across two locked axes. Reasonableness is judged against the mode presets and, for beta, the peer distribution.

3 · Wiring model (the connectedness contract)

Single source of truth: the seven slider inputs (g, m, t, rf, bt, mrpS, termg). Everything else is derived on every render():

sliders ─▶ cur() ─▶ modelFull() ─▶ { fair value, EV, value split, projection, CAGR }
  │              ├─▶ formula grids (β, MRP, CoE, WACC, TV) — numbers light up in place
  │              ├─▶ tornado impact ranking ─▶ sets sensitivity + reverse default axes (top-2)
  │              ├─▶ sensitivity 1-var / 2-var (centered on selection)
  │              └─▶ reverse DCF (implied value + 2-var grid + reasonableness)
modes/chips are just quick-sets of the sliders → highlight when the slider matches

Rules. (1) Modes never hold a value independent of a slider — clicking a chip sets the slider; the chip highlights only when the slider matches. (2) Tab/axis order follows impact ranking everywhere. (3) All data tables freeze the top row and left column (Excel-style). (4) Any "read" (a conclusion) is designed to feed the step below it — the beta context feeds the reverse-DCF beta verdict, etc.

5 · Instructions for use

1. Set a scenario (or drag the three business sliders). 2. Choose beta & MRP methods — watch the CAPM/WACC formulas recompute. 3. Read the value-breakdown bar: how much rides on the perpetuity. 4. Check the projected-path CAGR for believability. 5. Use the tornado to see what matters, then the 2-var matrix on those two. 6. Flip to Reverse DCF to see what the current price is assuming and whether that's reasonable vs the modes and the peer betas.
